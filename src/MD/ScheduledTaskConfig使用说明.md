# ScheduledTaskConfig 定时任务配置类使用说明

## 概述

`ScheduledTaskConfig` 是一个Spring Boot定时任务配置类，主要用于定时清理本地过期的临时封面文件。该类通过Spring的`@Scheduled`注解实现定时任务功能，确保系统磁盘空间的合理使用。

## 功能特性

### 1. 核心功能
- **定时清理**：每日凌晨2点自动执行临时文件清理任务
- **过期检测**：仅清理超过24小时的临时文件和目录
- **智能清理**：通过`WebScreenshotUtils.cleanupTempFiles()`实现核心清理逻辑
- **异常处理**：完整的异常捕获和日志记录机制

### 2. 技术特性
- **Spring集成**：使用`@Configuration`和`@EnableScheduling`注解
- **日志记录**：使用`@Slf4j`注解提供完整的日志功能
- **Cron表达式**：`0 0 2 * * ?` 配置每日凌晨2点执行
- **非阻塞设计**：异常不会影响后续定时任务的执行

## 类结构

### 注解说明
```java
@Configuration      // Spring配置类
@EnableScheduling   // 启用Spring定时任务功能
@Slf4j             // Lombok日志注解
```

### 核心方法

#### `cleanupExpiredTempFiles()`
- **功能**：定时清理过期的临时封面文件
- **执行时间**：每日凌晨2点
- **返回值**：无
- **异常处理**：捕获所有异常并记录日志

## 使用方法

### 1. 自动启动
该定时任务会在Spring Boot应用启动后自动开始工作，无需手动干预。

### 2. 配置要求
确保以下条件满足：
- Spring Boot应用已启用
- `WebScreenshotUtils`类可用
- 临时目录权限正常

### 3. 监控方式
通过应用日志监控定时任务执行情况：
```
=== 开始执行定时清理过期临时文件任务 ===
定时清理任务执行完成，共清理 5 个过期文件/目录，耗时 120 毫秒
=== 定时清理过期临时文件任务结束 ===
```

## 配置参数

### Cron表达式详解
```
0 0 2 * * ?
│ │ │ │ │ │
│ │ │ │ │ └─ 星期几 (? = 不指定)
│ │ │ │ └─── 月份 (* = 每月)
│ │ │ └───── 日期 (* = 每日)
│ │ └─────── 小时 (2 = 凌晨2点)
│ └───────── 分钟 (0 = 0分)
└─────────── 秒 (0 = 0秒)
```

### 过期时间阈值
- **默认值**：24小时
- **配置位置**：`WebScreenshotUtils.cleanupTempFiles()`方法内部
- **修改方式**：可通过重载方法`cleanupTempFiles(int expireHours)`自定义

## 注意事项

### 1. 系统要求
- **操作系统**：支持Windows、Linux、macOS
- **Java版本**：JDK 8及以上
- **Spring Boot版本**：2.0及以上

### 2. 性能考虑
- **执行时间**：选择凌晨2点避免业务高峰期
- **资源占用**：清理过程占用较少系统资源
- **并发安全**：单线程执行，避免并发问题

### 3. 安全注意
- **权限检查**：确保应用对临时目录有读写权限
- **路径安全**：仅清理指定的临时目录，避免误删
- **异常隔离**：异常不会影响应用主要功能

### 4. 维护建议
- **日志监控**：定期检查清理任务执行日志
- **磁盘监控**：监控临时目录磁盘使用情况
- **性能调优**：根据实际情况调整过期时间阈值

## 扩展功能

### 1. 自定义过期时间
如需修改过期时间阈值，可以在`WebScreenshotUtils`中调用：
```java
// 清理超过48小时的文件
WebScreenshotUtils.cleanupTempFiles(48);
```

### 2. 修改执行时间
修改`@Scheduled`注解的cron表达式：
```java
@Scheduled(cron = "0 0 3 * * ?")  // 改为凌晨3点执行
```

### 3. 添加清理统计
可以扩展方法返回更详细的清理统计信息。

## 故障排除

### 常见问题

1. **定时任务不执行**
   - 检查`@EnableScheduling`注解是否存在
   - 确认Spring Boot应用正常启动
   - 查看应用日志是否有相关错误

2. **清理效果不佳**
   - 检查临时目录路径是否正确
   - 确认文件权限设置
   - 调整过期时间阈值

3. **性能问题**
   - 监控清理任务执行时间
   - 考虑调整执行频率
   - 优化临时文件存储结构
